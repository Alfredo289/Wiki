A .md file that serves as the bridge between an Obsidian vault and Claude (or other LLM agents). Contains high-level personal context: who you are, your big question, how you like to work, main goals, current problems, team context.

Described by YB (yb-effect-2026-commonplace-practice) as the first file agents read before doing any work, ensuring alignment with the user’s current priorities and thinking style.

Key characteristics:

  • Living document - updated weekly as priorities shift
  • Team-like communication - treat it as briefing a team member, not configuring software
  • Portal function - the primary interface between personal knowledge and AI agents
  • Feeds subroutines - all knowledge-work-prs start by reading this file

Functionally similar to a CLAUDE.md project instruction file, but scoped to personal identity and goals rather than codebase conventions.

Relationship to Folder-as-Agent

Kieran Klaassen (klaassen-2026-folder-is-the-agent) describes a related pattern at the project level: the CLAUDE.md file in a project folder, combined with accumulated documentation and operational memory, turns the folder itself into an agent. Where the agentic constitution captures personal-level context (identity, goals, work style), the folder-as-agent captures project-level context (code conventions, architecture, runbooks). Both converge on the principle that accumulated context is the agent.

Claude Cowork “Brain File”

JJ Englert (englert-2026-claude-cowork-tutorial) demonstrates the same pattern in Claude Cowork: a preferences document (the “brain file”) that Claude reads at the start of every session. Contains who you are, how you work, communication style, and priorities. This is the agentic constitution applied within Cowork’s project-based UI, confirming the pattern extends beyond developer-oriented tools to mainstream productivity use cases.

Three independent sources now describe variations of this pattern: personal vault context (YB), project folder context (Klaassen), and Cowork project preferences (Englert).

Counter-arguments

  • Weekly updates are a significant maintenance burden for unclear returns
  • Risk of AI over-fitting to stated goals and filtering out serendipitous discoveries
  • The distinction from a standard system prompt or CLAUDE.md is unclear

Data gaps

  • No example of what a good agentic constitution looks like (YB references a template in replies but content not captured)
  • Now referenced by three sources (YB, Klaassen, Englert) across personal, project, and productivity contexts
  • No data on how much constitution detail actually improves agent output quality
