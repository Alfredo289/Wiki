A macOS productivity philosophy where the keyboard is the primary input device, minimizing mouse/trackpad usage through layered utilities. Described by Brett Terpstra as “the most powerful tool we have when using our Macs.”

The key insight is that the power comes from combining specialized tools into an ecosystem, not from any single utility. Terpstra’s stack covers six categories:

  1. Navigation - screen text search (Superkey), Vim bindings in any app (KindaVim), fuzzy menu search (Paletro)
  2. Key modification - system-level remapping (Karabiner-Elements), Hyper key setup, DefaultKeyBindings.dict
  3. Launching - app/file launcher (LaunchBar), static key sequences (LeaderKey)
  4. Text expansion - snippet-based (TextExpander), AI-predictive (Cotypist)
  5. Window management - keyboard-driven positioning (Moom)
  6. Web browsing - Vim navigation (Vimium), keyboard-first search (Kagi)

Overlaps with the folder-as-agent pattern in one respect: both are about accumulating a personalized environment over time that compounds in effectiveness.

Counter-arguments

  • High upfront learning curve and configuration cost
  • Tool overlap creates maintenance burden
  • Some users may get equivalent results with fewer, more integrated tools (e.g., Raycast replacing LaunchBar + clipboard + snippets)

Data gaps

  • No structured comparison of this stack vs simpler alternatives
  • Cost and conflict information missing
  • Single source
