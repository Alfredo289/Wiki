A workflow pattern where LLM agent subroutines process vault content and propose changes that the human reviews, analogous to code pull requests.

Described by YB (yb-effect-2026-commonplace-practice) as the culmination of the Claude + Obsidian setup, but only effective if the preceding practices (commonplace-practice, atomic-notes, intentional-reading, agentic-constitution) are in place.

How it works

  1. User defines subroutines (instruction sets) in the vault
  2. Each subroutine starts by reading the agentic-constitution
  3. Agent executes the subroutine and drafts changes
  4. User reviews changes one-by-one in a voice session (approve/tweak/reject)
  5. Approved changes get committed to the vault

Example: “Tend the Vault” - checks daily commonplace-practice entries since last run, identifies threads and insights, drafts updates to atomic-notes, presents changes for review.

The metaphor: agents are developers, you are the engineering manager reviewing PRs before they hit production.

Relationship to Folder-as-Agent

Kieran Klaassen's folder-as-agent pattern describes a similar human-in-the-loop dynamic at the engineering level: parallel Claude Code sessions work when each agent has a clear task and a human directs the work, but agent swarms coordinating autonomously fail. Knowledge Work PRs formalize this same constraint for personal knowledge management - the agent proposes, the human disposes.

Counter-arguments

  • The PR metaphor may break down - knowledge work changes are harder to diff than code
  • Heavy reliance on agent quality - poor drafts waste more review time than they save
  • Voice review sessions add friction compared to just writing the notes yourself

Data gaps

  • No subroutine examples shown (only “Tend the Vault” described at high level)
  • No data on approval/rejection rates or time savings
  • Klaassen’s folder-as-agent article provides indirect corroboration of the human-review pattern
