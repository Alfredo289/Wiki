Summary

Caveman Compress is a Claude Code skill that reduces token consumption by compressing project memory files (CLAUDE.md, todos, preferences) into a more concise format while preserving their semantic content and structure. Part of a broader pattern of context-compression tools that reduce the cost of persistent context in LLM workflows.

How It Works

  1. File type detection (local, no API call)
  2. Claude-powered compression (single API call to rewrite concisely)
  3. Validation (local checks that headings, code blocks, URLs, file paths, bullets survive)
  4. Targeted repairs (patches only broken sections, no full recompression)
  5. Output generation (compressed version + human-readable .original.md backup)

Preservation Rules

Code blocks, inline code, URLs, file paths, technical terms, heading text, table structure, and numeric values are never modified. Only natural language prose gets compressed.

Benchmarks

~46% average token reduction across five file types:

  • Preference files: 59.6%
  • Project notes: 53.3%
  • Project metadata: 43.3%
  • Todo lists: 38.1%
  • Mixed documentation: 36.9%

Real example: 706-token preferences file → 285 tokens, identical instructions.

Relationship to Other Tools

Addresses the same problem as Claude-Mem (reducing token costs for persistent context) but from the opposite direction: caveman compresses the files themselves, while claude-mem provides progressive disclosure to avoid loading everything.

Counter-arguments

  • 46% compression on small files saves little in absolute terms - CLAUDE.md files are typically 1-3K tokens
  • Compressed files may be harder for humans to read and maintain
  • Claude-powered compression introduces a dependency on the model it’s trying to optimize for
  • Validation catches structural breaks but can’t verify semantic preservation

Data gaps

  • No A/B testing of whether compressed vs uncompressed CLAUDE.md produces equivalent agent behavior
  • Benchmarks are from five files - unclear if gains hold across diverse projects
  • No discussion of failure modes when compression alters meaning
  • Python 3.10+ requirement limits accessibility
