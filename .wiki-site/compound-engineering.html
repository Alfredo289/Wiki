An AI-native engineering philosophy created by Kieran Klaassen at Every.to. The core idea: context accumulated in project folders over months of work makes each subsequent AI interaction more effective. The folder-as-agent pattern is a direct product of this approach.

Has 14,000+ stars on GitHub. Every.to offers a compound engineering guide and a plugin.

Efficiency Counterpart: Context Compression

As compound engineering accumulates context, token costs grow. Context Compression tools address this: Caveman Compress compresses the files themselves (~46% reduction), while Claude-Mem adds progressive disclosure to avoid loading everything (~10x savings). These are the efficiency layer that makes sustained compound engineering economically viable.

Counter-arguments

  • “Accumulated context improves results” is intuitive but hard to measure
  • May conflate project maturity with compound engineering benefits

Data gaps

  • Source article mentions compound engineering but doesn’t explain the methodology in detail
  • The GitHub guide and plugin not yet ingested as primary sources
  • No comparison with non-compound approaches on the same project
