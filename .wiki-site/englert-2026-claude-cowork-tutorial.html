Summary

A comprehensive walkthrough of Claude Cowork, Anthropic’s desktop tool positioned between simple chat and terminal-based Claude Code. Aimed at non-engineers who want AI to do work, not just suggest it.

Core Patterns

Brain file strategy - Create a preferences document that Claude reads at the start of every session. Contains who you are, how you work, communication style, and priorities. Functionally identical to an agentic-constitution, applied within Cowork’s project structure.

Projects as folders - Each Cowork project maps to a folder on your computer. Shared memory persists across conversations within a project, building context over time. Reinforces the folder-as-agent pattern: the project folder accumulates context that makes Claude more effective with each interaction.

One-click connectors - Gmail, Slack, Notion, Google Calendar connect with a single click. Permission controls let you set Claude to draft-only (never send) or always-ask mode. The connectors transform Claude from advisor (“here’s what you should do”) to executor (“I did it for you”).

Sub-advisory board - Spin up three AI agents with different personas (e.g., your boss, your engineering partner, your customer) to review work from multiple perspectives before you present it. Each gets a fresh context window for objectivity.

Scheduled tasks - Set up recurring tasks like a morning debrief at 7:30 a.m. that reads email, Slack, and calendar to produce a daily briefing. Moves AI from reactive (you ask) to proactive (it runs on schedule).

Progressive trust - Start with restrictive permissions, gradually expand as you build confidence. The tradeoff: more information access yields more productivity, but requires trust in the tool’s judgment.

Key Quotes

  • “Even though you might not be a developer, you’re now an AI orchestrator.”
  • “In chat it’s like cool, you told me what to do, but now I got to go do it. Co-work is literally doing it for you.”
  • “All my friends are agents. I’m a solo founder.” (on using sub-advisory boards)

Tools Referenced

Claude Cowork, Claude Code, Obsidian, Notion, OpenClaw, Wispr Flow, Monologue

Counter-arguments

  • Cowork’s value depends heavily on connector quality - if Gmail/Slack integrations are flaky, the whole workflow breaks
  • The sub-advisory board technique may produce artificial consensus rather than genuinely diverse perspectives
  • Scheduled tasks reading email and Slack raise significant privacy and data exposure concerns
  • “Progressive trust” could be a euphemism for gradually surrendering oversight

Data gaps

  • No comparison of Cowork vs Claude Code for overlapping use cases
  • No discussion of cost/token implications for scheduled tasks running daily
  • Sub-advisory board technique is described but no examples of actual output quality shown
  • How does Cowork handle errors or hallucinations in automated workflows?
