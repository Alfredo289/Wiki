A design pattern where a project folder - containing instruction files (CLAUDE.md/AGENT.md), skill definitions, documentation, and accumulated operational context - transforms a generalized LLM into a specialist agent. Coined by Kieran Klaassen in the context of building Cora at Every.to.

Core claim: An agent is a model with enough context so you don’t have to re-explain everything each time you open the chat. The folder provides that context. The same model pointed at different folders becomes a fundamentally different agent.

Layers of a Folder-Agent

  1. Conventions - coding standards, deploy workflows, naming patterns (CLAUDE.md)
  2. Institutional knowledge - architecture docs, design decisions, domain documentation
  3. Operational memory - runbooks, incident investigations, resolved bugs
  4. Specialized sub-agents - refined instruction files for specific tasks (reviewers, planners)

Relationship to Other Patterns

  • Agentic Constitution - similar principle applied at the personal level rather than project level. The constitution captures identity and goals; the folder captures project knowledge.
  • LLM Wiki Pattern - another form of pre-compiled context, but organized as a queryable knowledge base rather than an agent instruction set.
  • Compound Engineering - Klaassen’s broader methodology; the folder-as-agent pattern is one outcome of sustained compound engineering practice.

Claude Cowork as Folder-Agent UI

JJ Englert (englert-2026-claude-cowork-tutorial) demonstrates the folder-as-agent pattern in Claude Cowork: each project maps to a folder on your computer, with shared memory persisting across conversations. The project accumulates context (brain files, skills, connector configurations) that make Claude progressively more effective - the same compound accumulation Klaassen describes, but packaged in a GUI for non-engineers.

Anti-pattern: Agent Swarms

Klaassen explicitly warns against agent swarms - multiple agents coordinating with each other without human direction. The failure mode: agents don’t have a speed limit, but the human reviewing their output does. Parallel sessions work only when each agent has a clear task and a human directs the work.

Counter-arguments

  • “Give the model good context” is a well-known principle - the folder framing adds packaging but not necessarily new insight
  • The 44-agent claim is unverified and the dispatch layer is undescribed
  • Agent swarm failures may reflect 2026 tooling limitations, not a permanent constraint

Data gaps

  • Source article clipping is truncated - dispatch layer details missing
  • No examples of what a minimal vs mature folder-agent looks like
  • No metrics on how folder-agent quality improves over time
  • Now corroborated by Englert’s Cowork tutorial, though in a different (GUI) context
