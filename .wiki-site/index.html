Concepts

  • agentic-constitution - A living .md file giving LLM agents high-level context about the user. Read before any task, updated weekly.
  • atomic-notes - Note-taking practice where each note captures one concept. Attributed to Andy Matuschak. Enables cross-domain thinking and LLM recall.
  • commonplace-practice - Daily habit of capturing observations and thoughts in unstructured notes. Feeds context to LLM agents over time.
  • compound-engineering - AI-native engineering philosophy by Kieran Klaassen. Context accumulates in project folders, making each AI interaction more effective.
  • folder-as-agent - A project folder with CLAUDE.md, skills, and accumulated context turns a generalized LLM into a specialist. The context is the agent, not the model.
  • intentional-reading - Deliberate content consumption: curate sources, read with active highlighting, convert into atomic notes. Uses Readwise.
  • keyboard-driven-workflow - macOS productivity approach combining navigation, remapping, launching, text expansion, and window management utilities.
  • knowledge-work-prs - Vault subroutines where agents draft changes and humans review like pull requests. Example: “Tend the Vault.”
  • context-compression - Strategies to reduce token costs of persistent LLM context. Two approaches: compress files (caveman-compress) or retrieve selectively (claude-mem).
  • llm-wiki-pattern - Knowledge management approach where an LLM pre-compiles raw sources into structured wiki pages instead of using RAG. Originated by Karpathy.
